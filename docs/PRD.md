## 1. 产品概述

`blog-butterfly-go` 是一个面向个人博客场景的内容管理系统，当前由三部分组成：

- **前台站点**：`web-vue`，Vue 3 + Vite 构建的前台站点，由 Nginx 托管
- **后台管理端**：`admin-vue`，Vue 3 + Vite 管理后台，用于文章、分类、素材管理
- **后端 API**：Go + Gin + Gorm 实现，提供登录、文章、分类、素材等接口

当前产品的核心目标是：

1. 支持博客内容的创建、编辑、发布与删除
2. 支持分类管理与基础运营看板
3. 支持图床素材上传、浏览、复制与删除
4. 将后台逐步升级为更适合“持续写作”的内容创作工作台

## 4. 当前后台主流程

### 4.1 登录流程
1. 用户输入用户名和密码
2. 调用 `/api/login`
3. 后端校验用户密码
4. 返回 JWT token
5. 前端将 token 保存到 `localStorage`
6. 后续后台请求携带 token

### 4.2 文章管理流程
1. 进入文章管理页
2. 后台加载文章列表
3. 用户可筛选状态、分类、关键词
4. 用户可打开弹窗进行新建或编辑
5. 编辑器中完成封面、标签、分类、正文编写
6. 保存后写入后端数据库

### 4.3 素材使用流程
1. 用户在素材页上传图片到七牛云
2. 后台刷新素材列表
3. 在编辑文章时：
   - 可为封面选择图床图片
   - 可向 Markdown 正文插入图片链接

### 4.4 草稿保护流程
1. 用户在编辑器中输入内容
2. 前端对多个字段进行防抖自动保存
3. 草稿写入浏览器 `localStorage`
4. 用户重新打开编辑器时可选择恢复本地草稿
5. 用户正式保存到服务器后，本地草稿被清理

## 6. 当前产品约束

### 6.1 已确认技术约束
- 前台是静态页面，不是 SPA 框架项目
- 后台前端是 **原生 HTML/CSS/JS**
- 后端使用 **Gin + Gorm + MySQL**
- 图片依赖 **七牛云**
- 自动保存草稿当前仅保存在浏览器本地，不在服务端持久化
- 当前部署方式以 Docker 镜像 + K8s YAML 为主

### 6.2 当前能力边界
以下能力当前未在代码中体现为成熟产品能力：

- 多人协作编辑
- 服务端草稿版本历史
- 图片搜索、标签化素材库
- 富文本编辑器
- 评论审核工作流
- 完整权限分级后台

## 7. 当前产品问题

虽然系统已具备可用的博客后台基础，但仍存在以下问题：

1. 前台 README 显示“以后不再维护”，与当前后台持续开发状态存在文档层面的割裂
2. 后端数据库连接信息仍是硬编码到 `config/database.go`
3. JWT 默认密钥存在回退默认值 `your-secret-key`
4. 七牛配置从 `config.ini` 读取，环境化程度有限
5. 图片接口返回结构和文章接口风格不完全统一
6. 草稿自动保存仅在浏览器本地，跨设备不可同步
7. 编辑器增强已实现，但尚缺少统一的总览型产品文档

## 8. 近期产品重点

结合当前代码状态，建议当前版本重点放在以下方向：

### P1：后台主线稳定性
- 保持 `admin-vue` 为唯一现役后台主线
- 提升编辑、保存、恢复、素材插图链路的稳定性

### P2：创作效率增强
- 持续优化 Markdown 编辑体验
- 提升图片选择和插入效率
- 优化草稿恢复提示与状态反馈

### P3：配置与运维规范化
- 数据库/JWT/七牛配置改为环境变量或更清晰的配置方式
- 补齐产品文档、设计文档、部署文档的一致性
