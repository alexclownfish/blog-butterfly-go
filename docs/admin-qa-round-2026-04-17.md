# 管理后台验收问题记录（本轮）

项目：`/root/blog-butterfly-go`
范围：后台文章编辑、分类管理、素材管理
日期：2026-04-17

## 已定位问题

### 1. 分类页增删后未同步文章页分类筛选项
- 文件：`frontend/admin/app.js`
- 现象：
  - 新增分类后，文章列表顶部的“分类筛选”下拉不会立即拿到新分类。
  - 删除分类后，文章页筛选项可能仍残留旧分类。
- 根因：
  - `addCategory()` / `deleteCat()` 只刷新了分类列表和 dashboard，没有像 `renameCategory()` 一样同步调用 `loadCategoryOptions(..., { syncFilter: true })`。
- 影响：
  - 用户切回文章页时会看到过期的分类筛选项，造成“后台操作已成功但筛选器没更新”的错觉。

### 2. 素材页缺少统一反馈容器，复制链接成功后无可见提示
- 文件：`frontend/admin/app.js`
- 现象：
  - 素材页点击“复制链接”后，函数会调用 `showFeedback()`，但页面内没有 `#page-feedback` 容器，成功提示实际上不可见。
- 根因：
  - `loadImagesPage()` 未渲染 `<div id="page-feedback">`，而 `copyUrl()` 依赖该容器显示成功反馈。
- 影响：
  - 用户会怀疑“到底复制没复制成功”，交互反馈缺失。

### 3. 素材页图片加载未统一走 `requestJson()`，错误与登录态处理不一致
- 文件：`frontend/admin/app.js`
- 现象：
  - 素材页加载图片使用原始 `fetch(...).then(res => res.json())`；当接口返回 401/非 2xx 时，行为与其他页面不一致。
- 根因：
  - `loadImageHistory()` 未复用已有的 `requestJson()` 封装。
- 影响：
  - 登录失效、服务异常时，错误处理和跳转逻辑不统一，容易出现静默失败或提示不稳定。

### 4. 素材批量上传未校验单文件上传结果，可能“部分失败却显示全部成功”
- 文件：`frontend/admin/app.js`
- 现象：
  - `uploadMultiple()` 对每个文件只执行 `fetch()`，但未检查 `response.ok`。
  - 某张图片上传失败时，整体流程仍可能显示“已成功上传 N 张图片”。
- 根因：
  - 代码只等待 Promise 完成，不校验 HTTP 成功状态，也没有汇总失败结果。
- 影响：
  - 成功提示不可信，用户会误以为素材都已上传完成。

## 本轮拟修复项
1. 分类新增/删除后同步刷新文章筛选分类选项。
2. 素材页补充统一反馈横幅容器。
3. 素材图片列表加载改为统一走 `requestJson()`。
4. 素材批量上传增加响应校验与失败汇总提示。
5. 复制链接优先使用 `navigator.clipboard`，失败时回退到 `execCommand`，并提供清晰反馈。
